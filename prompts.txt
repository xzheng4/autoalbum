# AutoAlbum 项目用户指令记录

## 初始项目需求

这是一个家庭相册的项目。这个项目分为 2 个子项目。

子项目 1: 家庭照片分析与入库
家庭照片集的分析，分析内容包括人脸识别，重复照片，图片中 OCR 值提取，图片 EXIF 信息的提取，图片拍摄日期，拍摄设备，拍摄地点，图片内容理解，图片类型分类等常用图片归类方式。
要求所有的（大于 50000 张图片）图片描述与分析信息都严格结构化输出并记录到 sqlite 数据库中。

照片分析的要求如下：
1. 要求人脸识别的模型先从同一个家庭成员的一些已知图片中提起人脸信息，并记录人名用于后续图片按人物分类。
2. 人脸识别的模型采用当前流行的，实用的模型。
3. 图片分析采用 vLLM 0.16.0 版本（系统已经安装）作为推理引擎，并采用模型 Qwen/Qwen3-VL-4B-Instruct-FP8 做图片内容理解，图片分类分析，与 OCR 信息提取。每张图片只推理一次。
4. 计算环境为：GPU RTX 5060Ti 16GB 显存。
5. 图片材料数量有 50000 张以上，存放在几个多级子目录中。
6. vLLM 的分析推理过程可以按 batch=1，2，4，8，16，32 等方式推理
7. 程序处理过程可以随时被用户中断，下次启动支持扫描未处理图片，支持新增图片，已经分析过的照片自动跳过。

子项目 2: 家庭照片 webpage 的展示
根据子项目 1 的分析工具完成处理以后，读取图片数据库。
创建一个相册管理子项目，该相册管理子项目是用 python 构建的 web 前端，可以快速浏览已经分类的家庭图片，可以模糊搜索想要的图片。

### 指令 16: 添加进度条
- 为 scanner.py 的 add_new_images_to_db 函数添加 tqdm 进度条
- 为 main.py 的 analyze_all、refresh_faces、refresh_vl_analysis 函数添加 tqdm 进度条

### 指令 17: 重构分析流程
将 analyze_image 拆分成三个独立阶段：
1. EXIF 处理 - 提取并更新图片 EXIF 信息
2. 人脸识别 - 识别人脸并登记到数据库
3. VL 图片理解与分析 - 使用 Qwen3-VL 进行图片内容理解

新增 refresh-exif 命令，修改 analyze 命令流程：
1. scan: 扫描所有图片并登记到数据库
2. refresh-exif: 更新所有图片的 EXIF 信息
3. refresh-faces: 更新所有图片的人脸信息
4. refresh-vl: 更新所有图片的 VL 分析信息

所有处理过程都带有 tqdm 进度条。

### 指令 18: 支持断点续传
analyze 命令每次中断后重启会重复处理所有图片，需要支持断点续传。
- 数据库 images 表添加三个独立处理标志：exif_processed, face_processed, vl_processed
- refresh-exif/refresh-faces/refresh-vl 默认只处理未完成的图片
- 添加 --force 参数用于强制刷新所有图片
- analyze 命令跳过已完成阶段，从中断处继续处理

### 指令 19: VL JSON 解析失败跳过
当 VL 分析的 JSON 解析失败时，跳过该图片，不修改 vl_processed 状态，以便后续重试。

### 指令 20: check_db.py 随机抽取功能
给 tools/check_db.py 增加随机抽取功能：
- --random 参数：从所有图片中随机抽取
- --vl-json 参数：从已处理 VL 的图片中随机抽取并显示 JSON 数据

### 指令 21: 修复 VL JSON 解析 - 长 OCR 内容处理
修复当 OCR 内容较长时 JSON 格式易被破坏的问题（"Unterminated string starting" 错误）。
要求对可能有 OCR 内容的输出使用 json.dumps() 进行转义，并加强验证逻辑。
- 添加 `_fix_json_string` 方法逐字符扫描并转义控制字符
- 添加 `_extract_fields_robust` 方法从破损 JSON 中提取字段
- 使用多层回退策略解析模型输出

### 指令 22: 限制 OCR 文本长度
在 ANALYSIS_PROMPT 中添加要求，限制 `ocr_text` 字段最大长度为 1024 tokens。
如果图片中文字内容超过限制，截取最重要的前 1024 tokens。
